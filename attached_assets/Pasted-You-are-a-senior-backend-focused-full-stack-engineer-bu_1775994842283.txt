You are a senior backend-focused full-stack engineer building a production-grade system.

Project: "Self-Moving Hiring Pipeline"

Objective:
Build a backend-heavy web application that manages a hiring pipeline with automatic waitlist promotion, inactivity decay, and full traceability. The system must operate without manual intervention once configured.

IMPORTANT:
This is a backend-focused system. The frontend must be minimal and only used to visualize system state.

========================
TECH STACK
==========

* Backend: Node.js + Express + TypeScript
* Database: PostgreSQL
* Frontend: React (Vite) + TypeScript
* Styling: Minimal (Tailwind or basic CSS)
* Testing: Jest
* No external queue/job libraries

========================
CORE FEATURES
=============

1. Job Management:

* Create job with fixed ACTIVE capacity

2. Application Flow:

* If capacity available → applicant becomes ACTIVE
* Else → applicant enters WAITLIST (FIFO queue)

3. Automatic Promotion:

* When ACTIVE applicant exits → next WAITLIST applicant is promoted automatically

4. Applicant Capabilities:

* View status: ACTIVE / WAITLIST / INACTIVE
* View queue position
* View full event timeline

5. Concurrency Handling (CRITICAL):

* Handle simultaneous applications safely
* Prevent overfilling capacity
* Ensure deterministic queue order

6. Full Audit Logging (CRITICAL):

* Every state transition must be recorded
* System must be fully reconstructable from logs (event-driven thinking)

7. Inactivity Decay System (MOST IMPORTANT):

* When promoted → applicant must acknowledge within time window
* If not:

  * Move back to waitlist with penalty position
  * Promote next applicant
  * Continue cascade automatically
